## 1. 網站概述

### 1.1 網站名稱
「菠德桌上遊戲空間」會員系統

### 1.2 網站類型
會員管理 + 服務預約 + 現場POS複合型網站

### 1.3 目標用戶
- **前台：** 桌遊店會員（消費者）
- **後台：** 桌遊店老闆／員工（管理者）

### 1.4 核心功能摘要
| 角色 | 可用功能 |
|------|----------|
| 會員 | 註冊登入、租借遊戲、預約空間、紀錄遊玩時間、查看優惠券與積分 |
| 管理者 | 會員管理、等級管理、商品管理、庫存管理、租借歸還處理、遊玩時間紀錄、快速消費POS、優惠券設定、積分規則設定、報表檢視、活動公告管理 |

## 2. 會員系統

### 2.1 會員等級

| 等級名稱 | 升級時數門檻 | 升級金額門檻 | 福利 |
|----------|-------------|-------------|------|
| 小鳳梨 | 0 小時 | 100 元 | 遊戲購買9.5折、場地費平日55元/hr、假日65元/hr |
| 金鑽鳳梨 | 50 小時 | 10,000 元 | 遊戲購買9折、場地費平日50元/hr、假日60元/hr |

- 管理者可在後台新增、修改、刪除等級（含名稱、條件、福利）
- 等級採「同時滿足時數與金額門檻後自動升級」原則
- 福利說明：遊戲購買折扣、場地費（小時計費）優惠
- **非會員**為系統識別用，現場顧客不登入系統消費，福利為原價計費
- **非會員消費同樣會紀錄在系統中**（由管理者幫忙建立訂單），只是不登入系統、不累積積分、無法升級，僅作為現場顧客的消費資料留存

### 2.2 會員資料欄位
- 姓名（必填）
- 電話（必填）
- Email（必填）
- 生日（選填）
- 註冊日期（系統自動產生）
- 累積遊玩時數（系統自動累計）
- 累積消費金額（系統自動累計）
- 當前等級（系統自動判定）
- 會員狀態（啟用/停用）

### 2.3 會員積分制度
- 積分僅適用於小鳳梨（含）以上會員
- 消費1元 = 1 點積分（比例可由管理者調整）
- 積分可折抵消費（1點 = 1元，或由管理者設定）
- 積分嚴謹對帳，異動有紀錄

## 3. 商品與服務

### 3.1 實體商品
- 商品類別：桌遊、零食、飲料（視店家需求調整）
- 支援圖片、名稱、說明、售價、成本（僅管理者可見）、庫存管理
- 庫存不足時提供提醒功能
- 進貨記錄（日期、數量、供應商可備註）

### 3.2 空間租借
- 小時計費，平日/假日不同價
- 會員根據等級有不同單價（見2.1會員福利）
- 管理者於後台建立訂單，選擇時段與使用的會員

### 3.3 遊戲租借
- 次數計費：一次一周（可借閱天數由管理者設定）
- 會員可至前台提出租借申請，管理者後台審核並歸還日
- 歸還時可續借

### 3.4 在店遊玩計時
- 方案A：小時計費（依會員等級優惠）
- 方案B：包日方案（4小時可玩到關店，視店家定義）
- 遊玩起訖時間由管理者後台紀錄
- 系統自動累計會員「累積遊玩時數」

## 4. 金流與優惠

### 4.1 交易方式
- 現場收款為主（管理者直接收取現金或轉帳）
- 轉帳對帳：會員轉帳後，管理者於後台確認付款

### 4.2 優惠券功能
- 管理者可新增優惠券（名稱、折扣金額/比例、適用範圍、發行數量、有效期限）
- 會員於現場消費時可套用優惠券（透過POS系統）
- 優惠券類型：滿千折百、單品折扣、場地費折抵等

### 4.3 積分折抵
- 會員消費可累積積分，結帳時可選擇使用部分或全部積分折抵

## 5. 後台管理功能

### 5.1 會員管理
- 檢視會員列表（姓名、等級、累積時數、註冊日期）
- 搜尋、篩選會員
- 編輯會員資料、調整等級（手動升級/降級）
- 停用/啟用會員

### 5.2 等級管理
- 新增/編輯/刪除會員等級
- 設定名稱、升級條件（門檻、福利內容）

### 5.3 商品管理
- 商品上下架、編輯（名稱、說明、價格、成本、圖片）
- 庫存查詢、低庫存提醒（低於設定值主動提示）
- 進貨記錄

### 5.4 訂單管理
- 檢視所有訂單（商品購買、空間租借、遊戲租借、遊玩紀錄）
- 篩選條件：日期範圍、會員、等級、付款狀態
- 每月訂單對帳

### 5.5 儀表板快速功能

#### 5.5.1 快速消費（POS）
- **位置：** 儀表板首頁醒目按鈕，一鍵啟動
- **流程：**
  1. 輸入電話號碼，系統快速搜尋會員；若無搜尋結果，則為非會員
  2. 選擇商品或服務項目（商品、空間租借、遊戲租借等）
  3. 選擇適用的折扣（會員等級折扣、優惠券、積分折抵、或其他優惠）
  4. 系統顯示應收金額
  5. 確認後寫入訂單，完成交易
- **紀錄欄位：** 消費者（會員或非會員）、購買品項、折扣內容、應收金額、建立時間
- **適用情境：** 會員出示會員資格，管理者現場完成結帳；或非會員現場購買

#### 5.5.2 現場遊玩紀錄
- **位置：** 儀表板首頁，獨立區塊，即時顯示所有進行中的遊玩記錄
- **名單顯示：** 一目了然看到目前所有現場遊玩中的消費者，含狀態標記（遊玩中／已完成未結帳）
- **流程：**
  1. 選擇會員或非會員，輸入遊玩開始時間（系統立即記錄）
  2. 消費者遊玩結束後，管理者輸入結束時間
  3. 系統自動計算總遊玩時數（**無條件進位**至小時）
  4. 系統根據會員等級計算應收金額（場地費）
  5. 確認後寫入訂單，完成交易，並更新會員累積遊玩時數
- **狀態標記：** 遊玩中、已完成（待結帳）、已結帳
- **重要提示：** 仍有遊客未結帳時，儀表板應有明顯警示

### 5.6 優惠券與積分設定
- 優惠券：新增、修改、發放、查看使用情形
- 積分規則：調整消費與積分兌換比例

### 5.7 報表功能（每月報表）
| 報表類型 | 內容 |
|----------|------|
| 營收報表 | 總營收、成本、獲利（毛利） |
| 暢銷商品排行 | 商品銷售TOP10 |
| 會員消費排行 | 會員消費金額TOP10 |
| 等級分布統計 | 各等級會員人數 |

- 支援匯出（PDF或CSV）
- 可自行選擇日期範圍

### 5.8 活動布告欄與行事曆
- 管理者可發布活動公告（比賽、優惠資訊等）
- 行事曆檢視，重要日期可標記

## 7. 非功能性需求

### 7.1 技術建議
- 使用 ASP.NET Core MVC（.NET 8，開源跨平台，微軟官方技術）
- 資料庫：SQLite（用戶量少足夠，無需額外安裝資料庫服務），未來可遷移至 SQL Server 或 PostgreSQL
- 前台響應式設計（支援手機操作）
- 會員密碼使用 BCrypt 雜湊儲存

### 7.2 效能需求
- 會員人數：100人
- 訂單量：每月500筆以內
- 系統應在一般個人電腦或小型雲端主機上順暢運行

### 7.3 介面要求
- **介面簡約、方便操作**（老闆導向）
- 後台避免過多術語，使用直覺命名
- 手機版檢視正常

### 7.4 免費使用
- .NET 為開源免費方案，ASP.NET Core 亦為開源
- SQLite 為免費開源資料庫
- 無任何授權費用

## 8. 部署架構規劃

### 8.1 部署方案

| 方案 | 適用情境 | 優點 | 缺點 |
|------|----------|------|------|
| **Render 免費方案** | MVP階段、中小規模 | 免費、易部署、自帶HTTPS、.NET 原生支援 | 免費版實例睡眠（建議付費升級避免休眠問題） |
| **DigitalOcean / Linode VPS** | 正式營運 | 穩定、費用低（約5-6美元/月）、可完整控制環境 | 需要基礎維運能力 |
| **Azure App Service** | 官方平台需求 | 微軟官方、與 .NET 整合最佳 | 免費版（F1）限制多，不建議正式營運使用 |
| **本地主機（店主自己的電腦）** | 初期展示、開發測試 | 零成本、即刻上線 | 需電腦常開機、外部網路需設定 port forwarding |

### 8.2 採用方案

**使用 Render 免費方案 + 免費子網域（onrender.com）**

- ASP.NET Core MVC + SQLite 架構，部署至 Render
- 預設網址：`https://bord-game-space.onrender.com`（示意，最終名稱待定）
- Render 自動提供 HTTPS / SSL 憑證
- 若未來需要自訂網域，可於 Render 後台另設定
- 備註：.NET 程式部署至 Render 時，建議使用 **自訂 GitHub 部署（Web Service）** 而非靜態網站，Render 會自動執行 `dotnet build` 與 `dotnet run`

### 8.3 資料備份

- SQLite 資料庫檔案（`.db`）每次部署時自動持久化（Render 磁碟保留）
- 建議每月手動匯出 CSV 留存（報表功能已含此功能）
- 重要資料可額外備份至 Google Drive / Dropbox

### 8.4 安全性建議

- 後台路徑加強驗證（登入頁 + 複雜密碼）
- 管理者密碼使用 BCrypt 雜湊儲存
- 定期更新 .NET SDK 版本（安全修補）
- Render 平台提供基礎 DDoS 防護，無需自行設定防火牆

## 9. 目前的限制與假設

1. **不整合物流：** 商品購買以到店取貨為主，暫不串接物流
2. **轉帳對帳：** 會員轉帳後，管理者需手動確認付款狀態（建議 Email 或 LINE 通知輔助）
3. **比賽/活動報名：** 目前先做公告功能，報名系統列為未來擴充
4. **員工管理：** 暫不区分老闆與員工權限，所有管理者功能共用

### 11.5 各階段驗收方向

> 老闆在每階段交付時，請依以下清單逐一測試，確認功能正確後再進入下一階段。

#### 第一階段交付：會員系統 + 商品管理

**會員系統驗收：**
- [ ] 新會員可以使用電話/Email/密碼成功註冊
- [ ] 會員可以使用電話/Email 登入，錯誤密碼會被拒絕
- [ ] 會員可查看自己的資料（姓名/電話/Email/生日）
- [ ] 會員可修改自己的資料（密碼除外）
- [ ] 等級：預設有「非會員」「小鳳梨」「金鑽鳳梨」三個等級
- [ ] 管理員可新增/編輯/刪除會員等級
- [ ] 管理員可停用/啟用會員
- [ ] 等級升級條件正確：累積遊玩時數達50小時自動升級為金鑽鳳梨

**商品管理驗收：**
- [ ] 管理員可新增商品（名稱/說明/售價/成本/圖片）
- [ ] 商品列表可看到所有商品含庫存
- [ ] 管理員可上架/下架商品
- [ ] 前台可瀏覽商品（會員視角）

---

#### 第二階段交付：快速消費 POS + 現場遊玩紀錄

**快速消費 POS 驗收：**
- [ ] 儀表板首頁有「快速消費」按鈕，一鍵進入
- [ ] 輸入電話號碼，可正確搜尋到會員；電話不存在則視為非會員
- [ ] 選擇商品後，系統正確顯示該會員等級適用的折扣
- [ ] 可選擇優惠券，並正確抵扣金額
- [ ] 可選擇使用積分折抵，系統正確計算
- [ ] 應收金額計算正確（原价 → 折扣 → 優惠券 → 積分折抵）
- [ ] 確認後訂單寫入資料庫，庫存正確扣減
- [ ] 非會員消費同樣建立訂單，但不累積積分

**現場遊玩紀錄驗收：**
- [ ] 儀表板首頁有「現場遊玩名單」，一目了然看到所有人狀態
- [ ] 可新增遊玩者（會員/非會員），選擇後立即記錄開始時間
- [ ] 仍在遊玩的消費者狀態顯示「遊玩中」
- [ ] 輸入結束時間後，系統自動計算時數（無條件進位到小時）
- [ ] 金額根據會員等級計算正確（小鳳梨平日55/假日65，金鑽平日50/假日60）
- [ ] 非會員以原價計算（平日65/假日？請確認店家定義）
- [ ] 完成後狀態改為「已結帳」，訂單寫入資料庫
- [ ] 會員的「累積遊玩時數」正確增加
- [ ] 有未結帳消費者時，儀表板有明顯警示

---

#### 第三階段交付：優惠券 + 積分 + 儀表板

**優惠券驗收：**
- [ ] 管理員可新增優惠券（名稱/折扣/適用範圍/數量/期限）
- [ ] 優惠券在期限內可正常使用，過期後不可使用
- [ ] 會員可查看自己持有的優惠券
- [ ] POS 結帳時可選擇並正確抵扣

**積分系統驗收：**
- [ ] 小鳳梨（含）以上會員消費後，積分正確增加（1元=1點）
- [ ] 會員可查看積分餘額與異動紀錄（哪筆訂單累積/使用了多少）
- [ ] POS 結帳時可選擇使用積分折抵，1點=1元
- [ ] 積分不為負數，余額不足時不可超額使用
- [ ] 非會員不享有積分

**儀表板驗收：**
- [ ] 今日訂單數量顯示正確
- [ ] 今日營收加總正確
- [ ] 低庫存商品有警示標記
- [ ] 快速消費、遊玩紀錄按鈕存在

---

#### 第四階段交付：租借 + 庫存管理 + 報表

**空間租借驗收：**
- [ ] 會員可提出空間預約申請（選擇日期/時段）
- [ ] 管理員可在後台審核，通過或拒絕
- [ ] 預約成功後，時段不可重複預定
- [ ] 空間費用根據會員等級正確計算

**遊戲租借驗收：**
- [ ] 會員可提出遊戲租借申請
- [ ] 管理員可審核借出，設定歸還日期（一週後）
- [ ] 會員可在到期前續借
- [ ] 逾期歸還有提醒
- [ ] 遊戲租借費用（次數計費）正確

**庫存管理驗收：**
- [ ] 商品庫存可查詢
- [ ] 庫存低於警示值時，管理員可看到提醒
- [ ] 管理員可新增進貨記錄（日期/數量/供應商）
- [ ] 進貨後庫存正確增加

**報表驗收：**
- [ ] 選擇日期範圍後，報表數字與訂單管理中可查到的數據一致
- [ ] 營收報表：總營收 = 所有訂單應收金額之和
- [ ] 成本 = 所有商品售出數量 × 成本之和
- [ ] 毛利 = 營收 - 成本
- [ ] 暢銷排行：TOP 10 商品依銷售數量排序，正確
- [ ] 會員消費排行：依會員消費金額排序，正確
- [ ] 等級分布：各等級人數加總等於會員總數

---

#### 第五階段交付：活動 + 行事曆 + 預約 + 匯出

**活動公告驗收：**
- [ ] 管理員可新增/編輯/刪除活動
- [ ] 活動包含標題、內容、日期時間
- [ ] 前台可看到活動列表與詳情
- [ ] 活動可設定為「報名中/已截止/已結束」

**行事曆驗收：**
- [ ] 可檢視月檢視/週檢視
- [ ] 活動日期在行事曆上正確標記

**預約系統驗收：**
- [ ] 會員可提交空間/遊戲預約
- [ ] 管理員收到通知，可審核通過或拒絕
- [ ] 預約成功後，時段鎖定不可重複

**匯出功能驗收：**
- [ ] 報表可匯出 CSV 格式
- [ ] CSV 內容與畫面上顯示的數字一致

### 11.6 注意事項

- **老闆驗收是關鍵：** 每個階段交付後，請老闆實際操作一遍，確認商業邏輯無誤後再繼續，避免最後才發現問題
- **資料庫遷移：** 第一階段確定的資料庫結構後，尽量避免大幅修改，否則需重寫遷移腳本
- **分期交付的優點：** 問題早發現早修正，節省時間

### 13.3 備註

- **升級邏輯：** 會員同時滿足「時數門檻」與「金額門檻」時自動升級（非會員除外）
- **非會員識別：** MemberId 為 null 者為非會員，姓名電話可不填
- **BCrypt：** 密碼一律使用 BCrypt 雜湊（cost factor 12）
